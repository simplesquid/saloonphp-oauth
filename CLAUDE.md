# saloonphp-oauth

Concurrent-safe OAuth auto-refresh for Saloon v4 and Laravel.

## Stack

- PHP ^8.4 (asymmetric visibility, `#[\Override]`, readonly value objects)
- Saloon v4
- Laravel ^12.0 || ^13.0 (via `spatie/laravel-package-tools`)
- Pest 4 with arch + laravel plugins
- Laravel Pint (laravel preset + strict types + alpha imports)
- PHPStan level 10 via Larastan

## Hard rules

1. `declare(strict_types=1);` at the top of every file.
2. Final classes by default; value objects use `final readonly`.
3. `#[\Override]` on every override/interface method implementation (not on trait method overrides in consuming classes -- PHP doesn't consider traits as parents).
4. Exceptions are non-final classes extending standard PHP exceptions (like the odata package).
5. No AI attribution anywhere -- no Co-Authored-By, no "generated by", no AI mentions in commits/code/docs.
6. No emojis in source or docs.
7. Commit messages: title-only, natural language.

## Architecture

Three traits, two contracts, one model, one migration, one config.

**Traits** (in `src/Concerns/`):
- `ResolvesOAuthDependencies` -- shared base trait with `resolveTokenStore()`, `resolveTokenLocker()`, `resolveExpiryBuffer()`, `isExpired()`.
- `HasAutoRefresh` -- for `AuthorizationCodeGrant` connectors. Overrides `defaultAuth()`. User must implement `resolveTokenKey(): string`.
- `HasClientCredentialsCache` -- for `ClientCredentialsGrant` connectors. Overrides `defaultAuth()`. Token key defaults to `static::class`.

Both main traits use `VoidAuthenticator` in the `requestModifier` to prevent recursive auth during token acquisition/refresh.

**Contracts** (in `src/Contracts/`):
- `TokenStore` -- get, put, revoke, forget. Uses Saloon's `OAuthAuthenticator` directly. `get()` and `put()` both throw `TokenRevokedException` on revoked keys -- `put()` refuses to un-revoke so concurrent refresh-vs-revoke races cannot resurrect a token. Callers must `forget()` before re-using a revoked key.
- `TokenLocker` -- distributed lock around refresh/acquire.

**Failure semantics in the traits**:
- Persist failures after a successful refresh are logged via `report()` and the fresh token is returned to the caller. The current request succeeds; the next request retries the refresh (which will fail if the provider rotated the refresh token -- unsolvable without idempotent refresh).
- `TokenRevokedException` from `put()` is propagated (not swallowed) so the current request fails, matching the intent of the revocation.

**Support** (in `src/Support/`):
- `EloquentTokenStore` -- backed by configurable model, encrypted tokens. Clears token values on revoke.
- `CacheTokenLocker` -- backed by `LockProvider` from cache store. Separate TTL and wait timeout.
- `NullLocker` -- no-op for testing.

**Testing** (in `src/Testing/`):
- `InMemoryTokenStore` -- shipped for consumers to use in their own tests.

**Auth** (in `src/Auth/`):
- `VoidAuthenticator` -- no-op `Authenticator`, prevents recursive auth on refresh/acquire requests.

**Exceptions** (in `src/Exceptions/`):
- `TokenRevokedException` -- thrown when loading a revoked token.
- `TokenAcquisitionFailedException` -- wraps failures during refresh or acquisition.
- `LockTimeoutException` -- could not acquire the distributed lock.
- `InvalidCacheStoreException` -- cache store does not support locking.

## Commands

```bash
composer test          # Pest
composer analyse       # PHPStan level 10
composer format        # Pint
```

## Testing conventions

- Unit tests: no Laravel, no database. In `tests/Unit/`.
- Feature tests: Testbench with SQLite, `RefreshDatabase`. In `tests/Feature/`.
- Test fixtures in `tests/Fixtures/` (connectors).
- Saloon `MockClient::global()` for mocking HTTP in trait tests.
- TOCTOU tests use anonymous class lockers that mutate the store before calling the callback.
